text: "People ask how this site works. Here is the full breakdown.\n\n**What it is:** A microblog built with Astro. No CMS, no database, no admin panel. Every post is a single markdown file with frontmatter (date, text, tags). The site is static HTML deployed to a self-hosted server.\n\n**How posts get published:** I type a slash command in Claude Code. That triggers a skill that writes the markdown file, picks tags, runs `astro build`, rsyncs the output to my server over SSH, and fixes file ownership. One command, zero browser tabs.\n\n**The stack:**\n- Astro (static site generator)\n- Marked (markdown rendering)\n- rsync over SSH (deploy)\n- VPS running Apache or Nginx (hosting)\n- Claude Code skill (publishing workflow)\n\n**How the skill works:** Claude Code supports custom skills, which are reusable prompt templates triggered with a slash command. The `/stack` skill takes raw text, cleans it up, generates the next sequential filename, writes the markdown, runs the deploy script, then commits and pushes to git.\n\n**Why this approach:** I wanted to post without friction. A CMS adds login screens, update prompts, plugin conflicts. A static site with a CLI publishing workflow means I can go from thought to published in under 30 seconds without leaving my terminal.\n\n**How to build your own:**\n1. Clone this repo\n2. Replace the placeholder content with your own\n3. Set up a deploy script pointing to your server\n4. Create a Claude Code skill that automates the workflow\n5. Type one command. Post is live.\n\nNo accounts to manage. No tokens expiring. No vendor lock-in. Just markdown, a build step, and a server you control."
